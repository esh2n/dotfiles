# Workday Calc — 勤務時間算出

Slack MCPを使って対象日のSlack activity を取得し、勤務時間を算出してファイルに保存する。
Workday への入力は `/workday-input` で行う。

## Prerequisites

Slack MCP が有効であること。

## 使い方

```
/workday-calc              # 今日の勤務時間を算出
/workday-calc 2026-03-31   # 指定日
/workday-calc week         # 今週(月〜金)をまとめて算出
/workday-calc last-week    # 先週(月〜金)
/workday-calc month        # 今月の営業日をまとめて算出
/workday-calc last-month   # 先月の営業日
/workday-calc 2026-03      # 指定月の営業日
/workday-calc entry        # (非推奨: /workday-input を使うこと)
```

## Step 1: 日付の解決

- 引数なし → 今日の日付
- `YYYY-MM-DD` → その日
- `YYYY-MM` → その月の営業日（月〜金、祝日は含めない）
- `week` → 今週の月曜〜金曜（土日はスキップ）
- `last-week` → 先週の月曜〜金曜
- `month` → 今月1日〜今日までの営業日
- `last-month` → 先月の全営業日
- `entry` → Step 5 にスキップ（算出済みレポートからプロンプト生成のみ）

対象が複数日の場合、以下の Step 1.5〜4 を各日に対して実行する。

## Step 1.5: 勤怠チャンネルの確認

`slack_read_channel` で勤怠チャンネル（ID: `G016P32SBRS`）を読み、対象期間のユーザーの申請を確認する。

```
channel_id: G016P32SBRS
oldest: {対象期間の開始 Unix timestamp}
latest: {対象期間の終了 Unix timestamp}
response_format: concise
```

レスポンスから `U08KZLBBS1W`（ユーザーID）を含むメッセージを抽出し、以下のパターンを検出する:
- `年次有給休暇（終日）` → その日を「休暇」としてマーク、勤務時間算出をスキップ
- `年次有給休暇（午前）` → 「午前半休」、開始時刻を 13:00 に設定
- `年次有給休暇（午後）` → 「午後半休」、終了時刻を 13:00 に設定
- `病欠` / `欠勤` → 「病欠」としてマーク、スキップ
- `在宅勤務` → 勤務地を「リモート」に（デフォルトなので通常変更不要）
- `早出・早上がり` → 備考として記録

この情報は Step 3 の算出時に優先的に適用する。

## Step 2: Slack activity の取得

Slack MCP の `slack_search_public_and_private` を使い、対象日の自分のメッセージを取得する。
ユーザーの user_id は検索クエリ内で `from:<@{user_id}>` として使う。

### 最初のメッセージ（昇順）

```
query: "from:<@{user_id}> on:{YYYY-MM-DD}"
sort: timestamp
sort_dir: asc
include_context: false
limit: 5
response_format: detailed
```

### 最後のメッセージ（降順）

```
query: "from:<@{user_id}> on:{YYYY-MM-DD}"
sort: timestamp
sort_dir: desc
include_context: false
limit: 1
response_format: detailed
```

### 全件数（concise で件数だけ取る）

```
query: "from:<@{user_id}> on:{YYYY-MM-DD}"
include_context: false
limit: 1
response_format: concise
```

→ レスポンスの `(N results)` から件数を読み取る。

### ギャップ検知（孤立メッセージの判定）

深夜帯のメッセージは一律除外しない（深夜残業の可能性があるため）。
代わりに、最初/最後のメッセージと他のメッセージ群との間に **3時間以上のギャップ** がある場合、そのメッセージは「孤立メッセージ」として扱う。

**例: 00:27 に1件 → 次が 10:08（9時間超のギャップ）**
→ 00:27 は孤立メッセージ → `slack_first` には 10:08 を採用
→ ユーザーに「00:27 のメッセージは孤立のため除外しました。含めますか？」と確認

**例: 23:00, 23:30, 0:15, 0:45（連続した深夜活動）**
→ ギャップなし → 全て稼働時間として扱う

最初のメッセージ取得で `limit: 5` としているのは、先頭が孤立メッセージの可能性があるため。

### メッセージが0件の場合

デフォルト値をそのまま使用し、`(Slack activity なし — デフォルト値を使用)` と注記する。

### 取得結果

- 最も早いメッセージの時刻（孤立除外後）→ `slack_first`
- 最も遅いメッセージの時刻（孤立除外後）→ `slack_last`
- メッセージ総数 → `slack_count`

## Step 3: 勤務時間の算出

### デフォルト値

| 項目 | 値 |
|------|-----|
| 開始 | 9:30 |
| 終了 | 19:00 |
| 休憩開始 | 12:00 |
| 休憩終了 | 13:00 |
| 勤務地 | リモート |
| 出社日 | 火曜 |

### 補正ロジック

```
start_time = min(9:30, slack_first)
end_time   = max(19:00, slack_last)
break      = 12:00-13:00 (固定)
location   = 火曜 ? "出社" : "リモート"
work_hours = (end_time - start_time) - 1:00(休憩)
```

- `slack_first` が 9:30 より早い → 開始を `slack_first` に（15分刻みに丸める）
- `slack_last` が 19:00 より遅い → 終了を `slack_last` に（15分刻みに丸める）
- どちらもデフォルト範囲内 → デフォルトのまま

### Workday エントリの分割（重要）

Workday では **1日の勤務時間を休憩で分割して2エントリで入力** する必要がある:

- **エントリ1（午前）**: start_time 〜 12:00
- **エントリ2（午後）**: 13:00 〜 end_time

例: 通常日（9:30-19:00）
→ エントリ1: 9:30-12:00 (2.5h) + エントリ2: 13:00-19:00 (6h) = 8.5h

例: 早出残業（8:15-21:00）
→ エントリ1: 8:15-12:00 (3.75h) + エントリ2: 13:00-21:00 (8h) = 11.75h

例: 午前半休（13:00-19:00）
→ エントリ1のみ: 13:00-19:00 (6h)

例: 午後半休（9:30-13:00）
→ エントリ1のみ: 9:30-13:00 (3.5h)

出力フォーマットもこの分割を反映する。

### 15分丸めルール

- 開始時刻: 切り捨て（9:07 → 9:00, 8:52 → 8:45）
- 終了時刻: 切り上げ（21:07 → 21:15, 19:33 → 19:45）

### 当日の扱い

対象日が今日の場合:
- 終了時刻は `(暫定)` と表記する
- 最終的な終了時刻は退勤後に再実行して確定する

## Step 4: レポート出力

### ターミナル出力

```
=== Workday Report ===

3/2 (月) リモート
  勤務: 9:30 - 19:00 (8.5h)
  休憩: 12:00 - 13:00
  Slack: 最初 9:35 / 最後 18:45 / 23件
  補正: なし（デフォルト範囲内）

3/17 (火) 出社
  勤務: 9:30 - 20:45 (10.25h)
  休憩: 12:00 - 13:00
  Slack: 最初 9:45 / 最後 20:39 / 45件
  補正: 終了 19:00→20:45
```

### ファイル保存

結果を `~/workday-reports/{YYYY}/{MM}.txt` に保存する。
ディレクトリがなければ作成する。既存ファイルがあれば上書き。

### フォーマット

```
3/2(月) 9:30-19:00 休憩12:00-13:00 リモート
3/3(火) 9:30-13:00 午後半休 出社
3/4(水) 病欠(年次有給休暇・終日)
3/17(火) 9:30-20:45 休憩12:00-13:00 出社
3/27(金) 休暇(年次有給休暇・終日)
```

### ユーザー確認

異常値（孤立メッセージ、Slack activity なし、極端な長時間勤務）がある日はフラグを立て、ユーザーに確認する。
確認が完了したらファイルを更新する。

## Step 5: 次のステップ

最後に「`/workday-input` で Workday に入力できます（`claude --chrome` で実行）」と案内する。
